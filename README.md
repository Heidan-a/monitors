## Client 说明

每个客户端都会记录一份 `connectConfig` 信息，用于描述与服务端的连接配置，包括：

- **address**: 服务端地址  
- **token**: 本机连接服务端时使用的令牌

### 配置文件与安全性

- 客户端在 **首次注册成功** 后，会将上述信息保存到一个本地 JSON 文件中。  
- **后续连接** 时，只要该文件存在，就可以直接读取并发起连接，而不再需要重新注册。  
- 服务端在处理接口请求时，实际使用的是 JSON 文件中的 `token` 等字段：
  - 如果本地文件被误改或恶意篡改（例如修改 `token`），请求将无法通过认证，从而起到一定的安全防护作用。
  - 可以进一步优化为：在 **每次连接成功后**，服务端回表校验一次 `token`，并在 **每次请求** 时进行比对，增强安全性。

### 注册流程与硬件信息更新

- `client` 的 `register` 接口会在 **请求头** 中携带服务端生成的 `token`，用于在服务端的 `client` 表中完成注册。
- 在服务端的 `ServerConfiguration` 类中，通过 `run` 方法，实现如下逻辑：
  - 每当有客户端主机连接到服务端时，都会调用一次 `updateBaseDetail`，
  - 用于更新该主机的最新硬件数据信息。
